# Тест-кейсы для API (Задание 2.1)

В этом файле описаны тест-кейсы для проверки 4-х эндпоинтов микросервиса объявлений (https://qa-internship.avito.com).

## 1. POST `/api/1/item` — Создание объявления

### Позитивные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| POST-01 | Создание валидного объявления со всеми полями | HTTP 200, в ответе возвращается `id` созданного объявления (строка или UUID). |
| POST-02 | Идемпотентность: Отправка двух одинаковых запросов подряд | Оба запроса возвращают HTTP 200, но создаются два разных объявления с разными `id`. |
| POST-03 | Создание объявления с ценой `price = 0` (например, товар отдают даром) | HTTP 200, объявление успешно создано. |

### Негативные проверки
| ID | Описание | Ожидаемый результат | Баг |
|---|---|---|---|
| POST-04 | Создание объявления без обязательного поля `name` | HTTP 400 Bad Request | Да |
| POST-05 | Создание объявления без обязательного поля `price` | HTTP 400 Bad Request | Да |
| POST-06 | Создание объявления с отрицательной ценой `price = -100` | HTTP 400 Bad Request | Да |
| POST-07 | Создание объявления с пустым именем `name = ""` | HTTP 400 / 422 Bad Request | Да |
| POST-08 | Несовпадение типов: Передача строки в поле `price` ("дорого") | HTTP 400 Bad Request | Нет |

---

## 2. GET `/api/1/item/:id` — Получение объявления по ID

### Позитивные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| GET_ID-01 | Получение существующего объявления по валидному `id` | HTTP 200, в теле ответа возвращается массив из одного элемента со всеми полями (`id`, `name`, `price`, `sellerId`, `statistics`). |

### Негативные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| GET_ID-02 | Запрос несуществующего объявления (рандомный сгенерированный UUID) | HTTP 404 Not Found. |
| GET_ID-03 | Передача некорректного формата `id` (например, число "12345" или простая строка вместо UUID) | HTTP 400 Bad Request. |

---

## 3. GET `/api/1/:sellerID/item` — Получение списка объявлений продавца

### Позитивные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| GET_SELLER-01 | Запрос объявлений для `sellerID`, у которого есть объявления | HTTP 200, возвращается массив объявлений, `sellerId` каждого объявления совпадает с запрошенным. |
| GET_SELLER-02 | Корнер-кейс (Изоляция): Объявления продавца А не появляются в списке продавца Б | HTTP 200, в списке продавца Б нет объявлений продавца А. |

### Негативные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| GET_SELLER-03 | Запрос с невалидным `sellerID` (например, строка "не-число" вместо числа) | HTTP 400 Bad Request. |

---

## 4. GET `/api/1/statistic/:id` — Получить статистику по item ID

### Позитивные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| STATS-01 | Получение статистики существующего объявления | HTTP 200, возвращается массив, где первый элемент содержит поля `likes`, `viewCount`, `contacts`, значения которых совпадают с переданными при создании предмета. |

### Негативные проверки
| ID | Описание | Ожидаемый результат |
|---|---|---|
| STATS-02 | Получение статистики по несуществующему `id` | HTTP 404 Not Found. |
| STATS-03 | Использование невалидного формата `id` (не UUID) при запросе статистики | HTTP 400 Bad Request. |

---

## 5. E2E (Сквозные сценарии)

| ID | Шаги | Ожидаемый результат |
|---|---|---|
| E2E-01 | 1. Создать объявление.<br>2. Вызвать GET `/api/1/item/:id`. | Данные из шага 2 (`name`, `price`, `sellerId`) полностью совпадают с отправленными на шаге 1. |
| E2E-02 | 1. Создать объявление для уникального `sellerId`.<br>2. Вызвать GET `/api/1/:sellerID/item`. | Сгенерированный `id` из шага 1 присутствует в списке из шага 2. |
| E2E-03 | 1. Создать объявление с определенной статистикой.<br>2. Вызвать GET `/api/1/statistic/:id`. | Полученная статистика полностью совпадает с той, что была отправлена при создании. |
